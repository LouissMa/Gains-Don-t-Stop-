# 测试说明文档（v3.0）

## 1. 超时功能如何测试

### 方式说明
- 本项目实现了 **定时检测（方式A）**：每小时自动扫描一次超时借用记录并生成违规记录。
- 同时，为了方便验收，访问违规相关页面也会触发一次检测（例如进入“违规管理/我的违规”页面）。

### 测试步骤（推荐）
1. 使用管理员账号登录：`admin / 123456`
2. 进入“借用审批”，选择任意一条申请执行：
   - 批准（pending -> approved）
   - 确认借出（approved -> borrowed）
3. 在数据库中将该条记录的 `end_date` 改为昨天（模拟超时）：

```sql
UPDATE borrow_apply
SET end_date = DATE_SUB(CURDATE(), INTERVAL 1 DAY)
WHERE apply_id = <你的借用记录ID>;
```

4. 打开“违规管理”页面（或“我的违规”页面），系统会自动检测并生成一条 `violation` 记录：
   - `violation_start_date = end_date + 1`
   - `violation_days` 为首次生成时的快照值
   - 页面会展示“当前实时超时天数”

## 2. 权限影响如何测试

### 规则
- 用户存在 `untreated` 违规记录时：禁止发起新的借用申请。

### 测试步骤
1. 确保普通用户 `user01` 存在 1 条 `untreated` 违规记录（通过上一步超时生成即可）。
2. 使用普通用户登录：`user01 / 123456`
3. 进入“我的申请” -> 发起新的借用申请提交
4. 系统应返回提示：`您有待处理的违规记录，无法申请借用设备`
5. 管理员进入“违规管理”，对该条违规执行“处理违规”（填写处理备注），状态变为 `processed`
6. 再次使用普通用户发起借用申请，应可正常提交（进入 pending）。

## 3. 归还与违规联动方案选择

### 选择方案
- 本项目选择 **方案B**：
  - 允许管理员直接执行“确认归还”
  - 若该借用记录已产生 `untreated` 违规记录，则归还完成后系统自动将违规记录标记为 `processed`，并写入处理备注：`设备已归还，系统自动处理`

### 验证步骤
1. 让某条借用记录处于 `borrowed` 且 `end_date < today`（存在超时）
2. 确认该记录已生成 `untreated` 违规（在“违规管理”能看到）
3. 管理员执行“确认归还”
4. 验证：
   - 设备状态变为 `available`
   - 违规记录状态从 `untreated` 自动变为 `processed`
