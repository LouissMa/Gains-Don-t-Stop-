# 《实验室设备借用管理系统》版本1.0：基础数据管理

## 技术栈

- 持久层：MyBatis
- 控制层：Spring MVC
- 视图层：JSP + JSTL + EL表达式
- 数据库：MySQL
- 项目构建：Maven

---

## 一、题目要求

### 1.1 背景说明

某高校实验室需要建立设备借用管理系统，实现设备和人员的基础数据管理，为后续业务功能提供数据支撑。

### 1.2 功能需求

**（1）设备管理模块**

管理员可对实验室设备进行增删改查操作，设备信息包括：

| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | INT | 设备编号，主键自增 |
| device_name | VARCHAR(100) | 设备名称，不可重复 |
| device_model | VARCHAR(100) | 型号 |
| category_id | INT | 所属分类ID，外键 |
| location | VARCHAR(100) | 存放位置（如：A楼301-柜3） |
| purchase_date | DATE | 购入日期（不能晚于当前日期） |
| status | VARCHAR(20) | 状态：available/borrowed/maintenance/retired |
| max_borrow_days | INT | 单次最长借用天数 |

普通用户可查看设备列表和详情（不可修改）。

**（2）用户管理模块**

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | INT | 用户ID，主键自增 |
| username | VARCHAR(50) | 用户名，唯一 |
| password | VARCHAR(100) | 密码（加密存储） |
| real_name | VARCHAR(50) | 真实姓名 |
| student_no | VARCHAR(20) | 学号/工号，唯一 |
| role | VARCHAR(20) | 角色：admin/user |
| phone | VARCHAR(20) | 联系电话 |
| email | VARCHAR(100) | 邮箱 |
| status | VARCHAR(20) | 状态：active/inactive |
| last_login_time | DATETIME | 最近登录时间 |

功能要求：
- 用户注册、登录、登出
- 登录验证（用户名+密码）
- 角色区分：管理员(admin) / 普通用户(user)
- 管理员可禁用/启用用户
- 修改个人信息

**（3）设备分类管理**

对设备进行分类管理，支持二级分类结构（如：电子仪器 → 示波器类），分类信息包括：分类编号、分类名称、父分类ID（parent_id，顶级分类为NULL）、分类描述。设备关联分类（一对多）。

**（4）输入合法性校验要求**

以下校验必须在Service层实现，不能仅依赖前端表单验证。Controller收到请求后先做基础格式校验（@Valid注解或手动判空），再由Service层做业务规则校验，校验失败统一抛出BizException并通过全局异常处理返回规范错误信息。

**用户模块**

| 操作 | 校验字段 | 校验规则 |
|------|---------|---------|
| 注册/新增用户 | username | 不能为空；长度4~50字符；只允许字母、数字、下划线；不能与已有用户名重复 |
| 注册/新增用户 | password | 不能为空；长度6~20字符 |
| 注册/新增用户 | real_name | 不能为空；长度2~50字符 |
| 注册/新增用户 | student_no | 不能为空；长度不超过20字符；不能与已有学号/工号重复 |
| 注册/新增用户 | phone | 若填写，必须符合手机号格式（11位数字） |
| 注册/新增用户 | email | 若填写，必须符合邮箱格式（含@和域名） |
| 注册/新增用户 | role | 不能为空；只能是 admin 或 user |
| 修改用户信息 | real_name | 不能为空；长度2~50字符 |
| 修改用户信息 | phone | 若填写，必须符合手机号格式 |
| 修改用户信息 | email | 若填写，必须符合邮箱格式 |
| 修改用户信息 | student_no | 不能为空；修改时需检查是否与其他用户重复（排除自身） |
| 修改密码 | old_password | 不能为空；需与数据库中当前密码匹配 |
| 修改密码 | new_password | 不能为空；长度6~20字符；不能与旧密码相同 |
| 禁用用户 | user_id | 不能为空；用户必须存在；若该用户有进行中的借用记录（status=borrowed），禁止禁用并提示 |
| 删除用户 | user_id | 不能为空；用户必须存在；若有任何借用记录（历史含在内），禁止删除，只允许禁用 |

**设备模块**

| 操作 | 校验字段 | 校验规则 |
|------|---------|---------|
| 新增设备 | device_name | 不能为空；长度不超过100字符；不能与已有设备名称重复 |
| 新增设备 | device_model | 不能为空；长度不超过100字符 |
| 新增设备 | category_id | 不能为空；对应分类必须存在 |
| 新增设备 | location | 不能为空；长度不超过100字符 |
| 新增设备 | purchase_date | 不能为空；不能晚于当前日期 |
| 新增设备 | status | 不能为空；只能是 available/maintenance/retired 之一（新增时不能直接设为 borrowed） |
| 新增设备 | max_borrow_days | 不能为空；必须为正整数；建议范围1~365 |
| 修改设备 | device_name | 不能为空；修改时需检查是否与其他设备重复（排除自身） |
| 修改设备 | purchase_date | 不能为空；不能晚于当前日期 |
| 修改设备 | max_borrow_days | 不能为空；必须为正整数 |
| 修改设备 | status | 只能是 available/maintenance/retired 之一；若设备当前 status=borrowed，禁止手动修改状态 |
| 删除设备 | device_id | 不能为空；设备必须存在；若有进行中的借用记录（status=borrowed），禁止删除 |

**分类模块**

| 操作 | 校验字段 | 校验规则 |
|------|---------|---------|
| 新增分类 | category_name | 不能为空；长度不超过50字符；同一父级下分类名称不能重复 |
| 新增分类 | parent_id | 若填写，父分类必须存在；不允许超过二级（即父分类本身不能有父分类） |
| 修改分类 | category_name | 不能为空；修改时需检查同级下是否重名（排除自身） |
| 修改分类 | parent_id | 若修改，不能将分类设为自身的子分类（防止循环引用） |
| 删除分类 | category_id | 不能为空；分类必须存在；若该分类下有设备，禁止删除；若有子分类，禁止删除（需先删除子分类） |

**（5）统一接口返回格式**

所有Controller接口必须返回统一的Result对象，格式如下：

```java
public class Result<T> {
    private Integer code;    // 200=成功，400=业务错误，500=系统错误
    private String message;
    private T data;
}
```

**（6）全局异常处理**

使用@ControllerAdvice实现全局异常拦截，统一捕获业务异常（BizException）和系统异常，返回规范的Result错误响应，不允许异常信息直接暴露到页面。

## 四、注意事项

- 密码必须加密存储（使用MD5或BCrypt）
- 登录后使用Session存储用户信息
- 前后端不分离，纯JSP渲染
- 所有Service层校验失败须抛出BizException，不得直接返回null或空字符串
- 本版本重点考察：SSM整合能力 + 基础CRUD规范性 + 数据库设计能力
