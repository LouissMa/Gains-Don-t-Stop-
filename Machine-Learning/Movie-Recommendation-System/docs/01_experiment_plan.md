# 实验方案（第1部分）

## 1. 作业目标

本项目只服务于《深度学习》期末大作业，目标是在60分评分标准下完成一篇结构完整、实验真实、结果可复现的小论文，不按毕业设计规模扩展。

暂定题目：

> 基于神经协同过滤的个性化电影推荐方法研究

核心任务：

> 根据用户历史高评分电影，预测用户对未观看电影的偏好，并生成Top-N个性化推荐列表。

任务输入为用户历史评分记录，任务输出为按预测偏好排序的电影列表。本实验不实现复杂的商业推荐系统，也不把登录、后台管理等普通系统功能作为主要内容。

## 2. 数据集

采用GroupLens发布的MovieLens 100K数据集。

原始数据包含：

- 100,000条评分记录；
- 943名用户；
- 1,682部电影；
- 评分范围为1至5分；
- 每条评分带有时间戳。

选择理由：

1. 数据真实、公开且可核验；
2. 是推荐系统研究中的经典数据集，便于与相关工作对照；
3. 数据规模较轻，使用CPU或当前RTX 4060 Laptop GPU均可完成三模型对比；
4. 同时包含用户、电影、评分和时间信息，能够完成完整实验。

MovieLens 100K的1,682部电影均出现在评分记录中，因此全部进入模型电影词表。

数据文件保留原始压缩包及官方MD5校验文件。任何预处理都通过代码完成，不手工修改原始评分。

## 3. 任务定义

实验采用隐式反馈Top-N推荐任务。

- 将评分大于等于4分的记录视为用户喜欢该电影，即正反馈；
- 评分低于4分的记录不作为正反馈；
- 模型学习用户与电影之间的匹配概率；
- 对候选电影预测分数并进行降序排序；
- 取前K部电影作为推荐结果。

选择隐式反馈任务的原因是：推荐列表关心用户是否会选择某部电影，而不是只预测具体评分数值。该任务也与GMF、MLP和NeuMF模型的二分类训练目标一致。

## 4. 数据划分

采用按时间顺序的留一法，避免随机划分导致未来交互进入训练集。

对每位满足条件的用户：

1. 最后一条正反馈作为测试集；
2. 倒数第二条正反馈作为验证集；
3. 其余正反馈作为训练集。

仅保留至少具有3条正反馈的用户，保证每位用户在训练、验证、测试阶段都有数据。

必须通过程序检查：

- 同一用户的训练、验证和测试正样本不重复；
- 测试时间不早于验证时间；
- 验证时间不早于训练集最后时间；
- 用户和电影内部编号连续；
- 原始数据文件不被修改。

## 5. 负样本策略

MovieLens只记录用户已经评分的电影，没有直接提供“用户不喜欢或不会点击”的样本，因此需要负采样。

训练阶段：

- 每条正样本随机采样4部该用户从未评分的电影作为负样本；
- 每个epoch重新采样；
- 负样本不得出现在该用户的任何已知交互中。

验证和测试阶段：

- 每个真实目标电影搭配99部该用户从未评分的电影；
- 使用固定随机种子生成候选集；
- 同一次正式实验中所有模型使用完全相同的候选集。

该方案属于采样排序评估，论文中必须如实说明，不能写成对全部电影进行排序。

## 6. 对比模型

### 6.1 GMF

广义矩阵分解（Generalized Matrix Factorization）分别学习用户和电影嵌入，通过逐元素乘积建模二者交互。GMF作为较浅的神经协同过滤基线。

### 6.2 MLP

多层感知机（Multi-Layer Perceptron）拼接用户嵌入和电影嵌入，通过多层全连接网络学习非线性交互。

### 6.3 NeuMF

神经矩阵分解（Neural Matrix Factorization）融合GMF的线性交互表示和MLP的非线性交互表示，作为本实验的主要模型。

三种模型来自同一神经协同过滤框架，能够在相同任务、数据划分和评价协议下公平比较。论文中不把GMF夸大为深层网络，而是明确称为浅层神经协同过滤基线。

## 7. 训练目标

- 标签：正样本为1，负样本为0；
- 损失函数：二元交叉熵损失；
- 优化器：Adam；
- 模型选择：以验证集NDCG@10最高的epoch作为最优模型；
- 正式参数在代码完成和检查后共同确认；
- 所有模型使用相同随机种子、训练样本规则和评价候选集。

## 8. 评价指标

主要指标：

- HR@10：真实目标电影是否进入前10名；
- NDCG@10：同时衡量是否命中及命中位置，作为模型选择的核心指标。

辅助指标：

- HR@5；
- NDCG@5；
- AUC（基于采样候选集）。

由于每位用户在测试集中只有一个正样本，Recall@10与HR@10数值相同，Precision@10等于HR@10除以10。论文可解释这一关系，不用重复堆砌等价指标。
