# 第3章 方法与实施方案

### 3.1 整体方案

本文设计的个性化电影推荐流程包括数据读取与检查、隐式反馈转换、数据集划分、负样本构造、模型训练、离线评价和Top-N推荐展示七个环节。首先读取MovieLens 100K中的用户评分、电影信息和用户信息，检查评分范围、缺失值、重复交互以及用户和电影编号的有效性；随后将高评分记录转换为正反馈，并按照时间顺序划分训练集、验证集和测试集；在训练阶段为正样本动态采样负样本，分别训练GMF、MLP和NeuMF模型；最后利用统一候选集比较模型性能，并使用最优模型对指定用户的全部未评分电影进行排序。

整个方案以用户编号和电影编号为主要输入，通过嵌入层学习二者的低维潜在表示，不使用电影简介、海报或演员等额外特征。这样既能突出神经协同过滤对用户-电影交互关系的建模能力，也能在有限计算资源下完成真实、可复现的对比实验。

### 3.2 数据预处理与数据划分

#### 3.2.1 数据读取与隐式反馈转换

MovieLens 100K原始数据包含100000条1至5分的评分记录、943名用户和1682部电影。本文读取评分记录中的用户编号、电影编号、评分和时间戳，同时读取电影名称及类型，用于最终推荐结果展示。为了保证数据可靠性，程序自动检查数据表是否为空、评分是否处于1至5分范围内、用户-电影评分是否重复，以及评分记录引用的用户和电影是否真实存在。

【Zotero插入引用：Harper and Konstan, 2016】

本文将评分大于或等于4分的交互定义为正反馈，表示用户对该电影具有较明显的兴趣。评分低于4分的记录不作为正样本，但仍被视为用户已经接触过的电影，负采样和实际推荐时均予以排除。只保留至少具有3条正反馈的用户，以保证每位用户都能够划分出训练、验证和测试记录。原始用户编号和电影编号被映射为从0开始的连续内部编号，以便嵌入层进行索引。

#### 3.2.2 按时间顺序划分数据

为避免随机划分使未来交互进入训练集，本文对每位用户的正反馈按照时间戳升序排列，并采用留一法划分数据。用户最后一条正反馈作为测试样本，倒数第二条作为验证样本，其余正反馈作为训练样本。若相同时间戳下存在多条记录，则使用电影编号维持稳定顺序。

验证集用于每轮训练后的模型选择和早停判断，测试集只用于最优模型的最终评价。程序同时检查三个集合之间不存在重复用户-电影对，并保证训练记录时间不晚于验证记录、验证记录时间不晚于测试记录。该划分方式更符合推荐系统根据历史行为预测后续兴趣的实际过程。

#### 3.2.3 负样本构造

MovieLens只记录用户已经评分的电影，没有直接提供未点击或不感兴趣样本，因此需要进行负采样。训练阶段，对每条正样本随机选取4部该用户从未评分的电影作为负样本，正样本标签为1，负样本标签为0。负样本在每个训练轮次重新生成，并使用“固定随机种子加当前轮次”的方式保证随机过程可复现。

验证和测试阶段不重新随机变化候选集。每位用户的一个真实目标电影与99部从未评分电影共同构成100个候选项目，并在固定随机种子下打乱顺序。GMF、MLP和NeuMF共享完全相同的验证候选集和测试候选集，以避免候选项目差异影响模型比较。需要说明的是，该过程属于采样排序评价，并非对全部1682部电影进行测试排序。

### 3.3 神经协同过滤模型

#### 3.3.1 嵌入表示

设用户集合为 \(U\)，电影集合为 \(I\)，用户 \(u\) 和电影 \(i\) 分别通过嵌入层转换为32维向量 \(\mathbf{p}_u\) 和 \(\mathbf{q}_i\)。嵌入向量在模型训练过程中自动更新，用于表示用户兴趣和电影潜在特征。三种模型都以用户编号和电影编号为输入，但使用不同方式构造用户-电影交互函数。

【Zotero插入引用：He et al., 2017】

#### 3.3.2 GMF模型

GMF使用用户嵌入与电影嵌入的逐元素乘积表示交互：

\[
\mathbf{z}_{ui}^{GMF}=\mathbf{p}_u\odot\mathbf{q}_i
\]

随后通过一个全连接输出层得到预测分数：

\[
\hat{y}_{ui}=\mathbf{w}^{T}\mathbf{z}_{ui}^{GMF}+b
\]

与固定内积相比，输出层能够为不同潜在维度学习权重。GMF结构较浅、参数较少，适合作为线性交互方向的神经协同过滤基线，但其表达复杂非线性偏好的能力有限。

#### 3.3.3 MLP模型

MLP首先拼接用户嵌入和电影嵌入：

\[
\mathbf{z}_{ui}^{0}=[\mathbf{p}_u;\mathbf{q}_i]
\]

拼接后的64维向量依次经过维度为64、32、16和8的全连接层，各隐藏层使用ReLU激活函数：

\[
\mathbf{z}_{ui}^{l}=ReLU(\mathbf{W}_{l}\mathbf{z}_{ui}^{l-1}+\mathbf{b}_{l})
\]

最后将8维隐藏表示映射为一个预测分数。多层非线性变换能够学习比逐元素乘积更复杂的用户-电影交互关系，但也增加了模型训练和参数优化的难度。

#### 3.3.4 NeuMF模型

NeuMF将GMF分支与MLP分支结合。两个分支使用相互独立的用户和电影嵌入：GMF分支计算32维逐元素乘积，MLP分支将嵌入拼接后经过64、32、16和8维隐藏层。随后将GMF的32维表示与MLP的8维输出拼接，通过最终输出层得到匹配分数：

\[
\hat{y}_{ui}=\mathbf{w}^{T}
[\mathbf{z}_{ui}^{GMF};\mathbf{z}_{ui}^{MLP}]+b
\]

该模型同时保留GMF对潜在线性交互的刻画能力和MLP对非线性交互的学习能力，是本文的主要研究模型。本文不直接假设NeuMF一定优于其他模型，而是通过统一实验协议验证三种结构的实际效果。

### 3.4 模型训练方案

三种模型的输出均为未经Sigmoid激活的原始分数logit。训练使用带logit的二元交叉熵损失，其形式为：

\[
\mathcal{L}=-\frac{1}{M}\sum_{(u,i)}
\left[y_{ui}\log\sigma(\hat{y}_{ui})+
(1-y_{ui})\log(1-\sigma(\hat{y}_{ui}))\right]
\]

其中，\(M\)为当前训练样本数量，\(y_{ui}\)表示正负样本标签，\(\sigma\)为Sigmoid函数。实现中采用PyTorch的BCEWithLogitsLoss，以提高数值稳定性。优化器为Adam，学习率设为0.001，批大小为4096，最大训练轮数为30。

每轮训练结束后在固定验证候选集上计算NDCG@10。若当前结果高于历史最佳值，则保存模型参数和优化器状态；若连续5轮没有提升，则提前终止训练。最终加载验证集NDCG@10最高轮次的检查点，再在测试集上评价一次，避免根据测试结果反复调整模型。

| 参数 | 设置 |
|---|---:|
| 嵌入维度 | 32 |
| MLP隐藏层 | 64、32、16、8 |
| 训练负样本数 | 每个正样本4个 |
| 评估负样本数 | 每位用户99个 |
| 最大训练轮数 | 30 |
| 批大小 | 4096 |
| 学习率 | 0.001 |
| 优化器 | Adam |
| 损失函数 | BCEWithLogitsLoss |
| 早停耐心值 | 5 |
| 随机种子 | 2026 |

### 3.5 评价指标

设测试目标电影在候选列表中的排名为 \(r\)。当 \(r\leq K\) 时，说明目标电影进入前K名，HR@K取1，否则取0。对全部用户求平均后得到总体命中率：

\[
HR@K=\frac{1}{|U|}\sum_{u\in U}\mathbb{I}(r_u\leq K)
\]

NDCG@K进一步考虑目标电影的排名位置：

\[
NDCG@K=\frac{1}{|U|}\sum_{u\in U}
\frac{\mathbb{I}(r_u\leq K)}{\log_2(r_u+1)}
\]

目标电影排名越靠前，NDCG得分越高。采样AUC表示真实正样本得分高于候选负样本的比例，分数相同时按0.5次胜出计算。本文报告HR@5、NDCG@5、HR@10、NDCG@10和采样AUC，并将NDCG@10作为模型选择的主要指标。由于每位用户只有一个测试正样本，本实验中的Recall@K与HR@K数值相同，因此不重复报告。

### 3.6 推荐生成与结果展示

离线指标用于比较模型性能，推荐生成功能则用于体现算法的实际输出。对于指定用户，程序首先取得其所有已评分电影并从候选集中排除，然后使用训练完成的模型对全部剩余电影分批计算匹配分数。模型按照分数从高到低排序，返回前10部电影，并将内部编号还原为MovieLens原始电影编号，同时关联电影名称和类型信息。

该过程与测试阶段的采样评价不同：测试阶段使用固定的100个候选项目保证模型比较效率与公平性；实际推荐展示对用户全部未评分电影进行排序。最终输出包含推荐名次、用户编号、电影编号、电影名称、电影类型和模型分数，能够直观展示不同用户对应的个性化电影列表。

### 3.7 方案可行性与可复现性

在数据方面，MovieLens 100K来源公开、规模适中，能够支持三种模型的完整训练；在算法方面，GMF、MLP和NeuMF均适用于用户-项目隐式反馈任务，模型复杂度能够在现有GPU或CPU环境中承担；在评价方面，时间留一法、固定负样本候选集和多项排序指标能够形成较完整的验证流程。因此，本方案在数据、模型、计算资源和实验周期方面具有可行性。

为保证可复现性，程序固定Python、NumPy和PyTorch随机种子，并保存正式配置、每轮训练日志、最优检查点、测试指标及图表。实验同时记录Python、PyTorch、CUDA和GPU环境信息，对原始数据文件和源码计算SHA-256哈希。正式输出目录默认禁止覆盖，论文中的指标与图表均直接来自保存的CSV和图像文件，从而降低误操作或人工修改结果的风险。
