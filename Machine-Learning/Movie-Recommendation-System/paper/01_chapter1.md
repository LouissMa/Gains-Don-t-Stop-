# 基于神经协同过滤的个性化电影推荐方法研究

## 第1章 任务定义与背景分析

### 1.1 研究背景

随着互联网视频平台和数字媒体资源的快速发展，用户能够接触到的电影数量不断增加。丰富的内容一方面扩大了用户的选择范围，另一方面也带来了明显的信息过载问题。面对数量庞大、类型多样的电影资源，用户往往难以在有限时间内找到符合个人兴趣的内容。传统的热门榜单面向所有用户提供相同结果，虽然实现简单，但无法反映不同用户在电影类型、演员、年代和叙事风格等方面的偏好差异。推荐系统通过分析用户历史行为，从候选内容中筛选可能感兴趣的项目，是缓解信息过载、提高内容发现效率的重要技术手段。

【Zotero插入引用：Raza et al., 2025；Gheewala et al., 2025】

电影推荐是推荐系统的典型应用场景。用户在电影平台中的评分、点击、收藏或观看行为能够在一定程度上反映其兴趣偏好，系统可以据此学习用户与电影之间的潜在关联，并生成个性化推荐列表。早期推荐方法主要包括基于内容的推荐、基于邻域的协同过滤和矩阵分解等。协同过滤不要求人工总结复杂的电影规则，而是利用不同用户之间或不同电影之间的交互模式完成推荐，因此具有较强的通用性。然而，传统协同过滤通常依赖相似度计算或线性内积，面对稀疏的用户-电影交互数据时，对复杂非线性关系的表达能力有限。

深度学习能够通过嵌入表示和多层神经网络自动学习数据中的潜在特征及非线性交互，为推荐系统提供了新的实现方式。神经协同过滤将用户编号和电影编号映射为低维向量，再利用神经网络学习用户与电影之间的匹配关系。其中，神经矩阵分解模型NeuMF融合了广义矩阵分解对线性交互的建模能力与多层感知机对非线性交互的学习能力，适合用于隐式反馈条件下的个性化排序任务。

【Zotero插入引用：He et al., 2017；Li et al., 2023】

### 1.2 任务定义

本文研究的具体任务是：根据用户已有的电影评分记录，学习用户对电影的潜在偏好，并从用户尚未评分的电影中生成Top-N个性化推荐列表。该任务属于基于隐式反馈的Top-N推荐任务，其重点不是预测用户将给出多少分，而是判断候选电影是否可能成为用户感兴趣的内容，并按照预测偏好进行排序。

本文使用MovieLens 100K公开数据集开展实验。该数据集包含943名用户对1682部电影产生的100000条评分记录，每条记录由用户编号、电影编号、1至5分的评分以及时间戳构成，属于结构化用户-项目交互数据。MovieLens数据集来源清晰、规模适中，并被广泛用于推荐算法研究，适合在有限计算资源下完成可重复的模型训练与对比实验。

【Zotero插入引用：Harper and Konstan, 2016】

为使显式评分数据适用于隐式反馈推荐任务，本文将评分大于或等于4分的记录视为用户对电影的正反馈。模型输入包括用户编号及候选电影编号，训练阶段还包括由交互记录构造的正负样本标签；模型输出为用户与候选电影之间的匹配分数。实际推荐时，首先排除该用户已经评分的电影，再使用训练完成的模型对其余候选电影进行评分，最后按照得分从高到低选取前N部电影作为推荐结果。

从深度学习任务角度看，模型训练可以表示为二分类学习问题：正样本标签为1，采样得到的未交互电影标签为0；从应用目标角度看，最终任务属于个性化排序问题，需要利用模型分数确定候选电影的先后顺序。因此，本文采用命中率HR、归一化折损累计增益NDCG和采样AUC评价推荐效果，其中NDCG@10作为模型选择的主要指标。

### 1.3 研究目的与意义

本文的研究目的不是开发包含用户注册、电影录入和后台管理等功能的普通信息管理系统，而是围绕深度学习推荐算法完成一套可验证的个性化电影推荐方案。通过在统一数据集、统一数据划分和统一评价协议下比较GMF、MLP与NeuMF三种模型，可以分析不同用户-电影交互建模方式对推荐性能的影响，并验证融合线性交互与非线性交互是否能够改善推荐结果。

从理论学习角度看，本研究将协同过滤、嵌入表示、多层感知机、二元交叉熵损失和Top-N排序评价等知识连接为一个完整实验流程，有助于理解深度学习模型在推荐任务中的输入表示、训练目标与评价方式。从应用角度看，模型能够根据不同用户的历史行为输出不同的电影列表，体现推荐结果的个性化特征。与面向所有用户的统一热门榜单相比，该方法能够更有针对性地帮助用户发现潜在感兴趣的电影。

此外，本文重视实验结果的真实性与可复现性。实验使用公开数据集，固定随机种子和评价候选集，保存训练配置、每轮训练日志、最优模型参数、测试指标、运行环境以及数据和源码哈希。论文中的表格和图像均来源于程序保存的原始结果，不对实验数据进行人工补写或修改，从而保证研究过程能够被检查和复现。

### 1.4 研究内容与任务边界

围绕上述任务，本文主要完成以下工作。第一，对MovieLens 100K评分数据进行读取、检查和隐式反馈转换，并按照时间顺序为每位符合条件的用户划分训练集、验证集和测试集。第二，实现GMF、MLP和NeuMF三种神经协同过滤模型，在相同训练条件下开展对比实验。第三，使用HR@5、NDCG@5、HR@10、NDCG@10和采样AUC评价模型性能，并结合训练曲线和测试结果分析模型差异。第四，使用表现最好的模型对用户未评分电影进行预测，输出包含电影名称、类型和模型分数的Top-10推荐列表，以展示算法的实际推荐过程。

本文将研究范围限定在基于用户历史交互的神经协同过滤方法，不引入电影海报、文本简介、演员信息等多模态特征，也不重点讨论新用户和新电影完全没有历史记录时的冷启动问题。实验评价采用一个真实测试电影与99个固定负样本组成的采样候选集，而不是声称在测试阶段对全部电影进行了统一排序。实际推荐演示则会排除指定用户已经评分的电影，并对全部剩余候选电影计算偏好分数。上述边界既与现有数据和计算条件相匹配，也保证论文始终围绕深度学习个性化电影推荐这一核心任务展开。

本文章节安排如下：第2章调研传统协同过滤、矩阵分解、神经协同过滤及近年来深度推荐系统的发展，并总结常用数据集和评价指标；第3章详细介绍数据处理方法、GMF、MLP与NeuMF模型结构、负采样策略、训练流程和实施方案；第4章给出实验环境、数据统计、训练过程、模型对比结果和个性化推荐案例，并对实验结果进行分析。
