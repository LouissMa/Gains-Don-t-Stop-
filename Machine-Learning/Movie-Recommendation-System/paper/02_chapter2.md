# 第2章 相关工作调研

### 2.1 传统协同过滤方法

协同过滤是推荐系统中应用较早且具有代表性的方法，其基本思想是利用用户群体产生的历史交互发现相似偏好，而不必依赖电影内容的人工标注。按照相似关系的计算对象，传统邻域方法可以分为基于用户和基于项目的协同过滤。基于用户的方法寻找兴趣相近的用户，再根据相似用户的行为进行推荐；基于项目的方法计算电影之间的相似性，并根据用户喜欢过的电影推荐相似项目。Sarwar等提出的基于项目协同过滤方法将用户-项目评分矩阵用于项目相似度计算，在一定程度上提高了推荐效率和可扩展性。

【Zotero插入引用：Sarwar et al., 2001】

邻域方法结构直观，但容易受到交互矩阵稀疏和相似度不稳定的影响。矩阵分解则把高维用户-项目矩阵分解为用户潜在向量和项目潜在向量，通过二者的内积预测偏好。Koren等系统总结了矩阵分解在推荐系统中的应用，指出潜在因子能够表示用户兴趣和项目属性，并可进一步结合偏置、时间变化等信息。矩阵分解具有较好的表示能力和计算效率，但其内积形式本质上属于较简单的线性交互，对复杂用户偏好的建模仍存在局限。

【Zotero插入引用：Koren et al., 2009】

### 2.2 深度学习推荐方法

深度学习的发展使推荐模型能够利用神经网络自动学习用户和项目的低维表示及非线性关系。现有深度推荐方法包括多层感知机、卷积神经网络、循环神经网络、图神经网络和自监督学习等类型，不同方法分别适用于一般协同过滤、序列行为、图结构关系或稀疏监督等场景。相关综述表明，深度学习推荐研究已经从单一评分预测逐渐扩展到Top-N排序、序列推荐、图推荐和多模态推荐，但模型结构越复杂，通常也对数据规模、计算资源和训练过程提出更高要求。

【Zotero插入引用：Li et al., 2023；Gheewala et al., 2025】

神经协同过滤是将深度学习用于用户-项目交互建模的代表性框架。He等指出，传统矩阵分解使用固定内积函数描述用户与项目之间的关系，限制了模型学习复杂交互模式的能力。为此，该研究提出神经协同过滤框架，并设计了GMF、MLP和NeuMF三种模型。GMF通过用户嵌入和项目嵌入的逐元素乘积推广矩阵分解；MLP将两种嵌入拼接后输入多层感知机，以学习非线性交互；NeuMF融合两条分支的表示进行预测，同时保留线性和非线性建模能力。这一框架直接面向隐式反馈推荐，与本文的电影Top-N推荐任务具有较高的一致性。

【Zotero插入引用：He et al., 2017】

除神经协同过滤外，DeepFM将因子分解机与深层网络结合，在无须手工设计组合特征的情况下同时学习低阶和高阶特征交互，说明融合不同层次的交互信息能够改善推荐或点击率预测效果。然而，DeepFM主要面向包含多个类别特征域的点击率预测场景，而本文数据主要由用户编号、电影编号、评分和时间戳组成，因此不将其作为正式对比模型。

【Zotero插入引用：Guo et al., 2017】

### 2.3 推荐系统前沿研究

近年来，推荐系统进一步向图神经网络、自监督学习、生成式推荐和大语言模型推荐等方向发展。图神经网络将用户和项目组织为图结构，通过邻居信息传播学习高阶协同关系，适合表达多跳用户-项目联系。对比自监督学习则通过构造不同数据视图和辅助训练目标缓解监督信号稀疏问题。与此同时，生成式模型和大语言模型开始被用于推荐解释、对话式推荐和内容生成，为推荐系统提供了更丰富的人机交互方式。

【Zotero插入引用：Gao et al., 2022；Jing et al., 2023；Wu et al., 2024；Wang et al., 2024】

这些前沿方法拓展了推荐系统的能力，但也可能需要复杂图结构、文本信息、更大训练数据或较高计算成本。最新研究还开始关注噪声、恶意交互和对抗扰动对推荐可靠性的影响，说明实际推荐系统除准确性外，还需要考虑鲁棒性、公平性、隐私和可解释性。本文的重点是完成一个规模适中、过程可复现的深度学习推荐实验，因此选择NeuMF作为主要模型，并将上述方向作为相关研究和后续改进方向，而不在当前实验中盲目增加模型复杂度。

【Zotero插入引用：Zhang et al., 2026；Raza et al., 2025】

### 2.4 常用数据集与评价指标

推荐算法通常使用公开用户-项目交互数据集进行验证，常见数据集包括MovieLens、Amazon Reviews和Yelp等。其中，MovieLens提供用户对电影的评分及时间戳，具有数据来源清晰、版本规模多样和便于复现实验等特点，被广泛用于协同过滤研究。MovieLens 100K包含100000条评分记录、943名用户和1682部电影，规模适合课程实验，也能够支持按用户划分训练集、验证集和测试集。本文因此选择该数据集，并通过时间顺序划分降低未来交互进入训练集所造成的信息泄漏风险。

【Zotero插入引用：Harper and Konstan, 2016】

推荐系统评价方式主要分为评分预测和Top-N排序两类。评分预测常采用均方根误差和平均绝对误差，关注预测评分与真实评分之间的数值差异；Top-N排序则更关注目标项目能否进入推荐列表以及所处位置，常用指标包括Precision、Recall、HR、NDCG、AUC和MRR。HR@K表示测试目标是否出现在前K名；NDCG@K在命中的基础上进一步考虑排名位置，目标越靠前，得分越高；AUC用于衡量正样本得分高于负样本的整体概率。由于本文采用每位用户保留一个测试正样本的协议，Recall@K与HR@K数值相同，因此选择HR和NDCG作为主要排序指标，并将采样AUC作为辅助指标。

### 2.5 本文方法定位

现有研究表明，传统邻域协同过滤具有较强可解释性，矩阵分解能够学习潜在特征，而深度模型进一步增强了非线性交互建模能力。图神经网络、大语言模型和生成式推荐代表了新的研究趋势，但其数据和计算要求超出了本文的任务边界。结合MovieLens 100K的数据特点与课程实验条件，本文选择GMF和MLP作为两种结构不同的神经协同过滤基线，以融合二者特征的NeuMF作为主要模型。三种模型使用相同的数据划分、负采样规则和评价候选集进行比较，最终不仅报告离线排序指标，还使用最优模型生成实际Top-N电影列表，从而验证方法是否真正完成个性化推荐任务。
